# PRD.md — Nyaya: Product Requirements Document

**Version:** 1.0  
**Platform:** Android  
**Stack:** Flutter + FastAPI + Gemini + Indian Kanoon API  
**Cost to user:** ₹0, always

---

## 1. Problem Statement

Indian law students need access to case law, AI summaries, and legal research tools to study effectively. Every existing tool that provides these features (SCC Online, Manupatra, CaseMine, LegitQuest) requires expensive subscriptions that students cannot afford. The free alternative — manually searching court websites — is slow, fragmented, and produces no AI assistance.

**Nyaya solves this completely, for free.**

---

## 2. Target Users

**Primary user:** Fourth-year+ law student at an Indian law college
- Studies constitutional law, criminal law, civil procedure, corporate law
- Prepares for moot court competitions
- Needs to cite cases in assignments and written submissions
- Uses an Android phone as their primary device
- Has unreliable internet (hostels, libraries)
- Cannot afford ₹5,000–₹50,000/year for legal research tools

## 3. Core Features (Phase 1 — MVP)

### 3.1 Case Search
- User types a query (keyword, case name, citation, party name)
- App sends query to backend → Indian Kanoon API
- Results show: case title, court, date, citation, one-line snippet
- User can filter by: court (SC / HC / District), date range, subject area
- Results are paginated (20 per page)

### 3.2 Case Detail View
- Full judgment text rendered with proper formatting
- Metadata header: court, bench, date, citation number, coram (judges)
- "Download for Offline" button — saves full text + triggers AI summary
- "Cite This" button — copies citation in standard Indian format
- Share button — share case title + citation via WhatsApp/email

### 3.3 AI Case Summary
- Triggered when user opens a case (auto) or taps "Summarise"
- If summary exists in local DB → load instantly, no API call
- If not → fetch from backend (Gemini), save to DB, display
- Summary sections:
  - Plain English Summary (3–4 sentences)
  - Key Legal Issues (bullet list)
  - Court's Holding / Decision
  - Important Precedents Cited
  - Area of Law tags

### 3.4 Offline Library
- All downloaded cases visible in "My Library" screen
- Shows: case title, court, date, area of law, file size
- Filter by court, area of law, date saved
- Storage meter showing total MB used
- "Clear Cache" button (with confirmation dialog)
- Cases load in <200ms from local DB

### 3.5 Recent Judgments Feed
- Home screen shows latest SC + HC judgments (refreshed daily)
- Pulled from Indian Kanoon's latest feed
- User can filter feed by court or subject area
- Tapping opens case detail

## 4. Power Features (Phase 2)

### 4.1 Natural Language Search
- User describes a legal situation in plain English
- Backend sends it to Gemini, which extracts: legal principles, relevant acts, key terms
- Extracted terms are used to build a smarter Kanoon search query
- Results ranked by relevance score
- Example input: "Police arrested my client without telling him why"
- Example extraction: `Art.22 · D.K. Basu guidelines · Sec.57 CrPC · preventive detention`

### 4.2 Visual Citation Graph
- Available on any case detail screen
- Fetches `cites` and `citedby` from Indian Kanoon API
- Renders an interactive node graph:
  - Center node = current case (gold)
  - Blue nodes = cases this case cites
  - Green nodes = cases that subsequently applied this case
  - Red/purple nodes = overruled or distinguished cases
- Tapping any node navigates to that case
- Graph is cached in local DB after first load

### 4.3 Inline Annotations
- User can highlight any text in a judgment in 3 colours:
  - Yellow = general highlight
  - Blue = key ratio/holding
  - Green = important precedent cited
- Long-pressing highlighted text opens a note popup
- User types a personal note — saved to local DB
- All annotations persist offline
- Export button: generate PDF with highlights and notes visible

### 4.4 Judge Analytics
- Accessible from a judge's name (tap any judge name in a judgment)
- Profile page shows:
  - Name, current court, notable roles
  - Total judgments (approximate, from Kanoon data)
  - Subject matter breakdown (bar chart: constitutional, criminal, civil, etc.)
  - Notable landmark judgments (manually curated top 5)
  - Ideological tendency (e.g., "Rights-expansive", "Textualist") — derived by Gemini from landmark judgments
  - PIL admission rate (approximate)
- Data is generated by Gemini from public judgment metadata
- Cached in local DB, refreshed monthly

### 4.5 Bare Acts Integration
- Separate tab in the app: "Bare Acts"
- Source: legislative.gov.in (free, public domain)
- User can search acts by name or section number
- Full act text displayed, searchable
- Acts can be bookmarked
- Relevant acts linked from case summaries (e.g., "IPC § 302" → tappable link to act)

## 5. Student Superpowers (Phase 3)

### 5.1 Moot Court Prep Mode
- User enters the moot proposition (text field, can be long)
- Selects their side (Petitioner / Respondent / Both)
- Gemini generates:
  - 5 strongest arguments for each side
  - Key cases supporting each argument
  - Counter-arguments to anticipate
  - Suggested rejoinders
- User can save the moot prep as a PDF
- Supports major moot formats: Jessup, NLS, SCC, NUJS

### 5.2 Ask AI About This Case
- Available on any open case (online or offline)
- Chat interface, scoped to the current judgment
- Gemini has the case summary + key excerpts as context
- User can ask questions like:
  - "What does para 47 mean in plain language?"
  - "Which judge wrote the strongest dissent and why?"
  - "How does this affect Section 66A IT Act cases?"
  - "What's the difference between this case and Maneka Gandhi?"
- Conversation history stored per case in local DB
- Max 10 turns per session (token budget management)

### 5.3 AI Drafting Assistant
- Standalone "Draft" screen — a simple rich text editor
- As user types, Gemini detects legal arguments being made
- Sidebar shows real-time case suggestions relevant to what's being drafted
- User taps a suggestion → citation inserted into draft in correct format
- Export draft as DOCX or PDF
- Auto-formatting: inserts "Hon'ble", proper citation formats, section references

## 6. Non-Functional Requirements

### Performance
- App cold start: < 2 seconds
- Search results: < 3 seconds (network dependent)
- Offline case load: < 200ms
- AI summary generation: < 8 seconds
- Annotation save: < 50ms (local DB only)

### Storage
- Average case (text + summary): ~100KB
- 100 downloaded cases: ~10MB
- App must warn user when offline storage > 500MB
- App must not crash when storage > 1GB

### Reliability
- App must never crash when offline
- All network calls must have timeout (10s) and retry (2x)
- If Gemini API is down → show cached summary or graceful error
- If Indian Kanoon is down → show cached results or graceful error

### Accessibility
- Minimum font size: 14sp (adjustable in settings)
- All interactive elements: minimum 48dp touch target
- High contrast mode support
- Screen reader compatible (semantic labels on all widgets)

### Security
- API keys never stored in APK — always fetched from backend
- No user login required for core features
- No personal data sent to any server except: search queries (anonymous)

## 8. Out of Scope (v1.0)

- iOS version (Android only for now)
- User accounts / login
- Cloud sync of annotations
- International case law (only Indian courts)
- Audio playback / text-to-speech
- Real-time collaborative drafting
- In-app purchase or monetisation of any kind
